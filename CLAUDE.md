# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

A Claude Code skill pack for local marketers and agencies. Includes tools for brand auditing, performance testing, prospect research, and automated proposal generation with PDF output.

## Skills

| Command | File | Purpose |
|---------|------|---------|
| `/setup` | `.claude/commands/setup.md` | First-run onboarding. Creates `config/agency-config.json` with agency details, services, and pricing. |
| `/brand-audit` | `.claude/commands/brand-audit.md` | Scans a website across 4 dimensions (voice, visual, SEO, conversion). Supports `--save-json`. |
| `/lighthouse-audit` | `.claude/commands/lighthouse-audit.md` | Runs Google Lighthouse via npx. Reports scores, Core Web Vitals, and top issues. Supports `--save-json`. |
| `/prospect-research` | `.claude/commands/prospect-research.md` | Builds a business dossier using web search. Single-target or prospecting mode. Supports `--save-json`. |
| `/proposal` | `.claude/commands/proposal.md` | Orchestrator. Runs all three audits, generates HTML proposal and PDF via WeasyPrint. |

## First Run

1. `pip install -r requirements.txt` (installs WeasyPrint and Jinja2)
2. Run `/setup` to configure your agency profile
3. Run `/proposal <url>` to generate your first proposal

## Config System

- `config/agency-config.json` — generated by `/setup`, stores agency profile and preferences
- `config/pricing-templates/*.json` — bundled pricing structures (website-redesign, monthly-retainer, one-time-audit, hybrid)
- Pricing templates use a shared schema: `{ template_name, tiers: [{ name, price, highlight, description, line_items }] }`

## Output Pipeline

Skills write JSON to `output/` when using `--save-json` or when invoked by `/proposal`.
The `/proposal` orchestrator generates: `output/proposal-[name].md`, `.html`, and `.pdf`.
PDF generation uses `scripts/generate_pdf.py` (WeasyPrint).
The `output/` directory is gitignored.

## Dependencies

- **Python 3.x** with WeasyPrint and Jinja2 (see `requirements.txt`)
- **Node.js / npx** for Lighthouse audits
- **Chrome or Chromium** for Lighthouse to run headless audits

## Voice & Style

All skills follow the same voice rules:
- Direct, no preamble. Lead with findings.
- Concrete language: quote text, cite pages, name elements.
- No corporate filler, no hedging.
- Do NOT use em-dashes. Use periods, commas, or colons.
- Do NOT use the "not X, but Y" contrast structure.
